# 1. Introduction

## v0.5 FINAL — SC2026: Science-Aware Hybrid Retrieval with Dimensional Conversion for HPC Data Discovery

---

AI agents are rapidly entering scientific computing environments. Autonomous systems now execute real wet-lab experiments [Coscientist, Nature 2023], write research papers end-to-end [AI Scientist, 2024; AI Scientist-v2, 2025], reduce beamline preparation time by two orders of magnitude [ALS, arXiv:2509.17255], and orchestrate cross-facility experiments at ORNL [SC'25]. MCP has emerged as the standard protocol for connecting these agents to HPC infrastructure: MCP Servers for Science and HPC [arXiv:2508.18489] wrap Globus Search, storage, and compute resources under a unified interface. Agentic retrieval has matured — Context-1 [Chroma, 2026] achieves 88% answer recall through multi-hop search, while HiPerRAG [Gokdemir et al., 2025] scales RAG to 3.6 million scientific articles.

Yet these systems share a fundamental limitation: they treat scientific data as plain text. The ScienceAgentBench [Chen et al., NeurIPS 2024] shows the best agent achieves only 32% on scientific tasks — with most failures traced to data handling, not reasoning. Agents can think scientifically; they cannot *find* scientific data. Scientific corpora encode three types of structured knowledge that general-purpose retrieval ignores, leading to systematic failures in data discovery.

**The dimensional quantity problem.** Scientific measurements are routinely expressed in different units: "200 kPa," "200000 Pa," and "0.2 MPa" describe the same pressure. Existing approaches fail in distinct ways. String-based quantity normalization [QFinder, Almasian and Gertz 2022; Numbers Matter!, Almasian et al. 2024] maps unit names to canonical strings but cannot match across SI prefixes — "kilopascal" and "pascal" remain different strings, so "200 kPa" never matches "200000 Pa." Learned embedding approaches [CONE, Shrestha et al. 2026; NC-Retriever, 2025] encode units in vector space but cannot guarantee cross-prefix equivalence — the Numeracy Gap study [Deng et al., EACL 2026] shows embedding models achieve only 0.54 accuracy on numerical content. Neither approach performs the arithmetic conversion (200 × 1000 = 200000) required for dimensional reasoning.

**The formula representation problem.** Mathematical expressions appear in diverse notations — "F = ma," "F = m · a," "F=m*a," "ma=F" — and no retrieval system normalizes or matches across these variations. Math-aware search engines [Approach0; SSEmb, CIKM 2025] match equation *structure* in LaTeX, but cannot connect formulas to the physical measurements they describe. No system combines formula matching with quantity-aware retrieval.

**The scattered storage problem.** HPC data lives across parallel filesystems, S3-compatible object stores, vector databases, and graph databases. Scientists cannot query across these backends with a single search. Federated RAG approaches [RAGRoute, Guerraoui et al. 2025; Chakraborty et al. 2025] route queries to relevant data sources, but lack domain-specific retrieval primitives — they federate generic text search, not science-aware search. Meanwhile, HPC provenance systems [PROV-IO+, IEEE TPDS 2024] track data lineage but provide no search interface over the captured metadata. No system indexes HDF5 attributes, NetCDF metadata, or Parquet schemas into a unified, NL-queryable search engine [HDF Clinic, 2025].

These three problems compound. A scientist asking an AI agent *"Find experiments where pressure was between 190–360 kPa across all our storage"* needs dimensional conversion (kPa → Pa arithmetic), federated search (filesystem + S3 + vector DB), and potentially formula context (which equation relates pressure to other variables). No existing system addresses even two of these simultaneously.

We present **clio-agentic-search**, a hybrid retrieval engine that introduces *science-aware retrieval operators* — a new class of retrieval primitives for scientific computing. Our contributions:

1. **Dimensional-conversion-based measurement retrieval.** We canonicalize measurements to base units using explicit multiplication factors (kPa × 10³ = Pa, km × 10³ = m, mg × 10⁻⁶ = kg), enabling cross-prefix numeric comparison that is *guaranteed correct by construction*. Unlike string normalization [Almasian et al., 2024] or learned embeddings [Shrestha et al., 2026], our approach performs arithmetic conversion — "200 kPa" is numerically compared to "200000 Pa" through multiplication, not string matching or learned similarity. We support numeric range queries, exact value matching with tolerance, and unit-only filtering.

2. **Unified formula normalization and measurement retrieval.** We normalize mathematical expressions (whitespace, superscripts, factor ordering) and integrate formula matching with dimensional operators as parallel branches in a single hybrid pipeline. No prior system combines these modalities. The pipeline executes BM25 lexical, dense vector, graph traversal, and science-aware branches in parallel, merging results through deduplication and heuristic reranking.

3. **Federated multi-namespace search with capability negotiation.** A connector-based architecture enables unified retrieval across heterogeneous HPC storage — local filesystems, S3, Qdrant, Neo4j, Redis — without data migration. The retrieval coordinator dynamically negotiates available capabilities per backend, executing only supported branches. Incremental indexing with SHA256 change detection enables efficient re-indexing of evolving corpora.

Together, these contributions establish science-aware operators as retrieval primitives that complement — rather than replace — existing lexical and dense retrieval. clio-agentic-search is implemented as an open-source Python system with FastAPI and CLI interfaces, backed by DuckDB, passing 136 of 138 automated tests.
