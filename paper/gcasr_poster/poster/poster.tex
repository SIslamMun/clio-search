  \begin{block}{Abstract}
    \justifying
    Scientific data search demands reasoning that does not scale:
    inspecting what each dataset contains, converting between unit
    conventions, refining queries when initial results fall short. AI
    agents can automate this, but the cost of running them is rarely
    examined. A single query can consume nearly a million tokens before
    producing an answer.

    \vspace{0.5em}
    We present CLIO Search, which implements an
    observe--decide--act--evaluate cycle for scientific data.
    Science-aware operators execute deterministic SI conversion and
    formula normalization alongside BM25 and dense search. CLIO inspects
    what metadata each corpus provides and adapts accordingly. Federated
    search spans heterogeneous HPC backends with agentic refinement.

    \vspace{0.5em}
    We compare three agentic paths over 10 discovery queries against the
    National Data Platform (NDP, 341 datasets from NOAA, NASA, ECMWF,
    and others) using \texttt{claude-sonnet-4-20250514}. An agent backed
    by CLIO reduces tokens by \textbf{52\%} and wall time by
    \textbf{91\%} relative to a native Claude agent, while answering
    all 10 queries correctly.
  \end{block}

  %----------------------------------------------------------------------

  \begin{block}{CLIO Search}
    \justifying
    CLIO Search implements an \textbf{observe--decide--act--evaluate}
    cycle for scientific data discovery:

    \begin{enumerate}
    \justifying
    \itemsep0.3em
      \item \textbf{Discover (Observe):} Profile each corpus---document
        count, measurement count, metadata density, unit domains---to
        understand what search strategies are viable.
      \item \textbf{Reason (Decide):} Select retrieval branches based on
        what the corpus provides. Skip the scientific branch if no
        measurements exist; skip vectors if no embeddings are indexed.
      \item \textbf{Transform \& Search (Act):} Execute four parallel
        branches---\textbf{BM25}, \textbf{dense vector} (MiniLM-L6-v2),
        \textbf{graph traversal}, and \textbf{scientific operators}
        (SI conversion across 58 units in 12 domains + formula
        normalization)---then merge with a scientific hard filter.
      \item \textbf{Execute (Evaluate):} Assess results and decide to
        \textbf{expand}, \textbf{narrow}, \textbf{pivot}, or
        \textbf{stop}. Rewritten queries re-enter the pipeline.
    \end{enumerate}

    \vspace{0.5em}
    The agent receives compact ranked citations---not raw catalog JSON.
    CLIO handles unit conversion, profiling, and ranking \emph{before}
    the agent sees the results.
  \end{block}

  %----------------------------------------------------------------------

  \begin{block}{System Architecture}
    \begin{figure}
      \centering
      \includegraphics[width=\linewidth]{images/clio/architecture.pdf}
      \caption{CLIO Search architecture. Connectors index data from
      heterogeneous repositories (filesystems, HDF5, S3, IOWarp, NDP).
      At query time, the Namespace Registry routes queries to capable
      branches; the Retrieval Coordinator executes lexical, vector,
      graph, and scientific branches in parallel; Merge \& Rerank
      enforces dimensional correctness and returns top-$K$ results.
      The agentic loop refines queries iteratively.}
    \end{figure}
  \end{block}

  \begin{block}{Three-Path Experiment}
    \justifying
    We compare three ways of answering the same \textbf{10 scientific
    discovery queries} (temperature, pressure, wind, radiation, soil
    moisture) against the \textbf{National Data Platform} (NDP,
    341~datasets from NOAA, NASA, ECMWF, and others), using
    \texttt{claude-sonnet-4-20250514} as the LLM:

    \vspace{0.5em}
    \begin{description}
    \justifying
    \itemsep0.3em
      \item[Path 1 --- Claude Native Agent:] Web search, file tools,
        and sub-agent spawning. No domain-specific search. Must find
        NDP datasets by searching the open web.
      \item[Path 2 --- NDP-MCP:] Catalog search via Model Context
        Protocol. Returns \emph{raw dataset JSON}---the agent must
        parse and re-rank in context.
      \item[Path 3 --- CLIO:] Agent queries CLIO Search, which has
        indexed and profiled all 341~NDP datasets. Returns compact
        \emph{ranked citations} ($\sim$2K tokens per query).
    \end{description}
  \end{block}

  %----------------------------------------------------------------------

  \begin{block}{Why CLIO Is Faster}
    \justifying
    \begin{enumerate}
    \justifying
    \itemsep0.3em
      \item \textbf{Smaller context:} CLIO returns $\sim$2K tokens of
        ranked citations per query versus 989K tokens of raw JSON.
        The native agent spawns 9 sub-agents (563K + 426K tokens);
        NDP-MCP floods the context with raw catalog JSON.
      \item \textbf{Fewer reasoning steps:} 9/10 queries resolve
        in a \emph{single invocation}. Corpus profiling, unit conversion,
        and ranking execute as deterministic local operations---the
        agent interprets results rather than navigating raw data.
    \end{enumerate}

    \vspace{0.5em}
    \begin{tcolorbox}[colback=blue!5,colframe=blue!50!black,title=Key Insight]
    \justifying
    The bottleneck in agentic search cost is not the number of tool
    calls but the \textbf{shape of the data} the agent sees. Compact,
    pre-ranked citations scale better than raw catalog JSON.
    Co-designing specialized retrieval with the agent preserves accuracy
    while halving cost.
    \end{tcolorbox}
  \end{block}

  %----------------------------------------------------------------------

  \begin{block}{Broader CLIO Results}
    \justifying
    Beyond agentic efficiency, CLIO achieves across controlled
    benchmarks, real observatory data, and distributed HPC:
    \vspace{0.3em}
    \begin{itemize}
    \justifying
    \itemsep0.3em
      \item \textbf{Cross-unit retrieval:} 0.99 P@5 vs.\ 0.40 for
        BM25 on 210 synthetic documents, 80 queries across 7 domains
      \item \textbf{Real-world NOAA data:} +87\% precision over hybrid
        baselines on 288 GHCN-Daily weather station records
      \item \textbf{Distributed scale:} p50 < 42\,ms across 2.5M arXiv
        documents on 4 NVIDIA GH200 nodes (NCSA DeltaAI), with 0.94
        weak-scaling efficiency
      \item \textbf{IOWarp integration:} Sub-linear profiling from 17\,ms
        (1K blobs) to 90\,ms (500K blobs) on IOWarp CTE storage
      \item \textbf{Federation:} 1.0 routing recall across 100 namespaces
        while skipping 52\% of irrelevant namespaces
    \end{itemize}
  \end{block}
