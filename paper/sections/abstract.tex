\begin{abstract}
Scientific data discovery requires search that thinks before it acts. Before executing a query, an effective search system must profile each corpus to understand what data and metadata exist. It must decide which search strategies apply, activating science-aware operators when structured measurements are present and falling back to text retrieval when they are not. It must execute domain-specific reasoning, such as the arithmetic needed to recognize that 200~kPa, 200000~Pa, and 0.2~MPa describe identical pressure. And when results are insufficient, it must refine its approach. This observe-decide-act-evaluate cycle is what distinguishes agentic search from conventional retrieval. Few current systems perform these operations in combination.

We present CLIO Search, an agentic retrieval system for scientific data. CLIO implements a four-stage pipeline: Discover (profile each corpus in 14--22\,ms), Reason (select retrieval branches based on corpus characteristics), Transform (execute dimensional conversion and formula normalization as deterministic database operators alongside BM25 and dense vector search), and Execute (federate across storage backends with iterative refinement). The refinement loop is driven by an LLM query rewriter in connected environments or a deterministic fallback for air-gapped HPC deployments; the fallback achieves 96\% of LLM-assisted quality. Domain knowledge is encoded as algebraic invariants in the retrieval operator, complementing learned similarity with arithmetic guarantees.

On cross-unit queries, CLIO achieves 0.99~P@5 where BM25 scores 0.40 and NC-Retriever achieves 0.16; on same-unit controls, all methods perform comparably (0.30--0.45), confirming gains are specific to cross-unit matching. Precision improves by 87\% on 288 real NOAA observatory documents. At distributed scale on 2.5M arXiv abstracts across 4~NVIDIA GH200 nodes, cross-unit queries return identical results at sub-50\,ms latency with 0.94 weak scaling efficiency. Integration with IOWarp blob storage achieves 0.5\% corpus inspection where the native API returns everything unfiltered. All code and benchmarks are open-source.
\end{abstract}
