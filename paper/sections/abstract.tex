\begin{abstract}
Scientific data discovery requires search that reasons, not just retrieves. When a climate scientist queries for pressures above 200~kPa, the search system must recognize that ``200000~Pa'' in an HDF5 archive and ``0.2~MPa'' in an S3 dataset are both matches. It must determine that one corpus has rich unit annotations while another has none, and adapt its strategy accordingly. It must search across filesystems, object stores, and binary formats without requiring the scientist to know where data resides. Current retrieval systems perform none of these operations: they match text, not meaning.

We present CLIO Search, an agentic retrieval system for scientific data that separates search reasoning from search execution. CLIO implements a four-stage pipeline: Discover (profile each corpus to learn what data and metadata exist), Reason (select retrieval branches based on corpus characteristics), Transform (execute science-aware operators including dimensional conversion and formula normalization as first-class retrieval branches), and Execute (federate across storage backends with iterative refinement). Unlike conventional pipelines that apply the same strategy to every dataset, CLIO inspects before it searches, adapts to what it finds, and encodes domain knowledge as deterministic computation rather than learned similarity.

CLIO achieves 0.99~P@5 on cross-unit queries where the strongest baseline (BM25) scores 0.40 and the best learned approach (NC-Retriever) achieves 0.16 dense retrieval accuracy. On 288 real NOAA observatory documents, precision improves by 87\% over hybrid baselines. Cross-unit correctness holds at distributed scale: on 70K arXiv documents across 4~NVIDIA GH200 nodes, queries in Pa, kPa, bar, and atm return identical results at sub-50\,ms latency. Integration with IOWarp blob storage demonstrates 0.5\% corpus inspection per query where the native API returns everything unfiltered. All code and benchmarks are open-source.
\end{abstract}
