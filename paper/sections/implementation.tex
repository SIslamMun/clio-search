\section{Implementation}

CLIO is implemented in Python 3.11+ (13{,}100 lines, 60 modules, 283 tests) and exposes a CLI and async HTTP API via FastAPI. All persistent state resides in a single DuckDB file (Table~\ref{tab:schema}), selected for embedded operation (no service to deploy on compute nodes), SQL-native full-text search for BM25, and single-file compatibility with POSIX parallel filesystems. The two science-aware tables store canonicalized SI values and normalized formula signatures, enabling cross-unit queries as direct SQL range predicates. Dense retrieval uses all-MiniLM-L6-v2 (384-dim) with exact or approximate (hnswlib) ANN backends.

Nine connectors implement the \texttt{NamespaceConnector} protocol with runtime-checkable capability declarations. The IOWarp connector bridges CLIO to tiered blob storage via CTE's \texttt{BlobQuery} and \texttt{GetBlob} APIs. The NDP connector discovers datasets from the National Data Platform via the Model Context Protocol. HDF5 and NetCDF connectors extract attribute-level units through the same SI canonicalization pipeline. Qdrant and Neo4j connectors are in-memory stubs that validate the protocol.

\textbf{Platforms.} Single-node experiments run on Linux 6.17 with DuckDB 1.1. Distributed experiments run on NCSA DeltaAI using NVIDIA GH200 Grace Hopper nodes (72~ARM Neoverse V2 cores, 480\,GB unified CPU+GPU memory, NVLink-C2C interconnect, Slingshot-11 network) with 2.5M arXiv abstracts across 4 DuckDB shards; a lightweight coordinator-worker architecture over TCP/JSON handles fan-out without MPI. IOWarp experiments run inside Docker (\texttt{iowarp/deps-cpu}, Chimaera v1.0.3, 16\,GB shared memory). The agentic efficiency experiment uses \texttt{claude-sonnet-4-20250514} via the Claude Agent SDK. The full source code, test suite, and evaluation scripts will be released.
