\section{Introduction}

Scientific computing generates data at unprecedented scale: a single exascale job on Frontier can produce 80~PB of simulation output. Yet an estimated 60 to 80\% of scientific data is never reused after creation~\cite{rag_numerical2025}, not because it is inaccessible, but because it is unsearchable. The FAIR data principles, now mandated by NSF, NIH, and DOE, require that data be Findable, Accessible, Interoperable, and Reusable. Automated FAIR assessments show that no major repository achieves even 50\% compliance on reusability, with measurement heterogeneity a systematic barrier. Nature's survey found that over 70\% of researchers have failed to reproduce others' experiments~\cite{baker2016reproducibility}. A central bottleneck is search.

This gap is intensifying as AI agents enter scientific workflows. Coscientist~\cite{coscientist2023} autonomously executes wet-lab synthesis. AI Scientist v2~\cite{aiscientistv2_2025} generates research end-to-end. Autonomous agents at ORNL orchestrate experiments across facilities~\cite{ornl_agents_sc25}. These agents can reason about science, plan multi-step experiments, and call external tools. What they cannot do is search scientific data effectively, because the retrieval infrastructure they depend on treats all content as plain text. ScienceAgentBench~\cite{scienceagentbench2025} reports 32.4\% end-to-end task success, with failures traced to data handling rather than reasoning. A comprehensive survey~\cite{agentic_ai_survey2025} identifies scientific data management as explicitly underexplored in the agent literature.

Current retrieval systems fall short of scientific data discovery in three fundamental ways.

First, scientific search requires domain reasoning over both numeric and symbolic content that retrieval systems do not perform. A query for pressures between 190 and 360~kPa must recognize that ``200000~Pa'' and ``0.2~MPa'' are both matches, but this requires arithmetic, not text similarity. Formulas compound the problem: ``$F = ma$,'' ``$F = m \cdot a$,'' and ``$ma = F$'' express the same physical law but produce distinct token sequences that defeat both keyword and embedding retrieval.

No existing approach closes this gap. The Numeracy Gap study~\cite{numeracygap2026} evaluates 13 embedding models and reports 0.54 binary retrieval accuracy on numerical content. String normalization~\cite{numbersmatter2024} maps unit names to canonical forms but cannot bridge SI prefixes. Learned approaches~\cite{cone2026,ncretriever2025} achieve up to 65\% recall improvement over baselines but provide no correctness guarantee. The consequences are not hypothetical: NASA's \$327M Mars Climate Orbiter loss (pound-seconds vs.\ newton-seconds), the Gimli Glider incident (kg vs.\ lb fuel loading), and an estimated 7,000 to 9,000 medication deaths per year from unit errors in US healthcare all stem from failures to reconcile equivalent measurements.

Second, scientific datasets vary enormously in metadata quality, and no search system adapts to this variation. Some datasets carry explicit unit annotations, quality flags, and provenance; others contain raw binary with no metadata at all. A fixed retrieval pipeline either wastes computation applying science-aware operators to datasets that have no extractable measurements, or misses opportunities when rich metadata is available. Effective scientific search requires inspecting each corpus to determine what strategies are viable before executing any query.

Third, scientific data is distributed across storage systems that no search engine spans with domain awareness. A single experimental campaign may deposit raw output on a parallel filesystem, preprocessed arrays in HDF5, derived features in a vector database, and catalog entries on an external platform. Existing systems either federate generic text search without science-aware operators~\cite{ragroute2025}, or capture storage provenance without exposing a search interface~\cite{provio2024}. Existing systems support unit normalization or federated access in isolation, but no system combines domain-aware retrieval with corpus-adaptive orchestration and federated access across HPC backends.

We present CLIO Search, a retrieval system that addresses all three challenges through a single design principle: separate search reasoning from search execution. Unlike conventional retrieval systems that apply a fixed pipeline to every query, CLIO reasons about how to search before searching. It profiles each corpus to discover what metadata exists (completing in 14--22\,ms), selects retrieval branches based on corpus characteristics (skipping branches that cannot contribute), executes science-aware operators as deterministic database predicates complementing BM25 and dense vector search, and federates across heterogeneous storage backends. An iterative multi-hop refinement loop, driven by an LLM query rewriter or a deterministic fallback that achieves 96\% of LLM-assisted quality, expands, narrows, or pivots the query based on intermediate results. Domain knowledge enters the pipeline as executable, verifiable computation: the fact that 5~km = 5000~m is an algebraic invariant encoded in the retrieval operator, guaranteed correct for all supported unit transformations.

We make four contributions:

\begin{enumerate}
\item \textbf{Science-aware retrieval operators.} We introduce dimensional conversion and formula normalization (algebraic equivalence matching, e.g., $F{=}ma$ vs.\ $ma{=}F$) as deterministic, first-class retrieval branches that execute alongside BM25 and dense vector search. A composable unit registry based on dimensional analysis canonicalizes measurements to SI base units at index time, enabling cross-unit queries as database range predicates with correctness guaranteed for all supported unit transformations (46~units, 12~domains). Adding a new unit requires one configuration entry, not code changes.

\item \textbf{Corpus-adaptive search orchestration.} A profiling layer inspects each namespace before querying, learning what metadata exists and which retrieval branches are viable. The system adapts its search strategy per dataset: rich-metadata corpora activate science-aware operators, sparse corpora fall back to text-based retrieval, and multi-hop refinement expands, narrows, or pivots the query based on intermediate results.

\item \textbf{Federated scientific search.} A connector architecture provides unified search across heterogeneous storage backends (filesystems, object stores, HDF5, NetCDF, IOWarp blob storage, external catalogs) with per-connector capability negotiation. Each connector declares which retrieval branches it supports, and the coordinator routes queries accordingly.

\item \textbf{Multi-scale evaluation.} We evaluate on a controlled benchmark (210~documents, 80~queries), real observatory data (288~NOAA GHCN-Daily documents), a live external catalog (341~NDP datasets), distributed HPC infrastructure (2.5M arXiv abstracts across 4~GH200 nodes on NCSA DeltaAI), and IOWarp blob storage (10K blobs). CLIO achieves 0.99~P@5 on cross-unit queries (vs.\ 0.40 for the strongest text baseline); on same-unit controls all methods score comparably (0.30--0.45), confirming the gains are specific to cross-unit matching. Real-world precision improves by 87\% on NOAA data, with sub-50\,ms query latency and 0.94 weak scaling efficiency at distributed scale.
\end{enumerate}
