# 1. Introduction

## 1.1 AI Agents Entering Scientific Computing

AI agents have moved beyond text generation into experimental science. Coscientist \cite{coscientist2023} autonomously plans and executes wet-lab synthesis through tool-augmented reasoning. AI Scientist v2 \cite{aiscientistv2_2025} generates, implements, and writes up research end-to-end. At the Advanced Light Source, an LLM-based agent reduces beamline preparation time by two orders of magnitude \cite{als_beamline2025}. ORNL deploys autonomous agents that orchestrate experiments across national computing facilities \cite{ornl_agents_sc25}, and the Model Context Protocol has been extended to wrap Globus search, compute, and storage under a unified agent interface for HPC \cite{mcp_hpc2025}. The trajectory is clear: agents are transitioning from processing scientific literature to operating scientific infrastructure.

Yet these agents consistently fail at data retrieval, not reasoning. ScienceAgentBench \cite{scienceagentbench2025}, a benchmark of 44 scientific tasks drawn from peer-reviewed publications, reports that the best agent achieves only 32.4\% end-to-end success; with expert-provided domain knowledge the figure rises to just 34.3\%, and failure analysis traces the dominant error mode to data handling. MLAgentBench \cite{mlagentbench2024} observes the same pattern: agents break at data preparation. A comprehensive survey of agentic AI \cite{agentic_ai_survey2025} identifies data management as an explicitly underexplored capability. These agents can reason about science. They cannot find the data they need to reason about.

The root cause is structural. Retrieval systems --- whether lexical, dense, or hybrid --- treat scientific content as plain text. Three compounding failures explain why this treatment is inadequate.

## 1.2 Three Compounding Failures

**Dimensional blindness.** Scientific measurements are routinely expressed in different unit prefixes: "200 kPa," "200000 Pa," and "0.2 MPa" describe identical pressure. No retrieval system equates them. Consider the three dominant approaches. String-based quantity normalization \cite{numbers_matter2024} maps unit names to canonical strings, but "kilopascal" and "pascal" are different strings --- the system cannot match "200 kPa" to "200000 Pa" because no string operation bridges the prefix gap. Learned embedding approaches such as CONE \cite{cone2026} encode numbers and units in a shared vector space, but the representation is probabilistic: the Numeracy Gap study \cite{numeracy_gap2026} evaluates 13 embedding models on numerical retrieval and finds a mean binary retrieval accuracy of 0.54 --- effectively random against a 0.50 baseline. NC-Retriever \cite{nc_retriever2025} reports 16.3\% dense retrieval accuracy on queries with numeric constraints. The fundamental problem is that none of these approaches performs the required arithmetic: $200 \times 1000 = 200{,}000$. Dimensional equivalence is a mathematical fact, not a similarity to be learned.

**Formula opacity.** Mathematical expressions appear in inconsistent notations across scientific documents. "F=ma," "F = m \cdot a," "F=m*a," and "ma=F" encode the same physical law, yet every representation produces a different token sequence. Math-aware search systems such as Approach0 \cite{approach0} and SSEmb \cite{ssemb2025} match equation structure in LaTeX markup, but they operate on formula syntax in isolation --- they cannot connect a matched formula to the physical measurements it governs. No retrieval system combines formula normalization with quantity-aware dimensional search.

**Storage fragmentation.** HPC scientific data is distributed across parallel filesystems, S3-compatible object stores, vector databases, graph databases, and binary scientific formats such as HDF5 and NetCDF. A single experimental campaign may span all of these backends. No existing system provides unified search with science-aware operators across them. Federated RAG systems such as RAGRoute \cite{ragroute2025} route queries to relevant data sources, but they federate generic text search --- none incorporates science-aware retrieval operators. Provenance systems such as PROV-IO+ \cite{provio_plus2024} capture detailed HDF5 I/O lineage with under 3.5\% overhead, but expose no search interface over the metadata they collect. The result: scientists resort to manual, per-backend searches that are both incomplete and unrepeatable.

These failures compound. Consider a scientist asking an AI agent: *"Find experiments where pressure was between 190--360 kPa across all our storage."* Answering this query requires dimensional conversion (kPa, Pa, and MPa must be compared arithmetically), federated search (the data spans filesystems, object stores, and HDF5 files), and potentially formula context (which equations relate pressure to other measured variables). No existing system addresses even two of these three requirements simultaneously.

## 1.3 Contributions

This paper introduces *science-aware retrieval operators* --- a new class of deterministic, pluggable retrieval primitives that execute as first-class branches alongside standard BM25 lexical and dense vector search. Unlike string normalization or learned embeddings, these operators encode domain knowledge as explicit arithmetic and symbolic transformations, making their correctness verifiable by inspection. We make the following contributions:

1. **Dimensional-conversion measurement retrieval.** We canonicalize physical quantities to base SI units via explicit arithmetic multiplication (e.g., kPa $\times$ $10^3$ = Pa, km $\times$ $10^3$ = m, mg $\times$ $10^{-6}$ = kg). The conversion is guaranteed correct by construction: if the SI conversion factor is correct, the match is correct. The operator supports numeric range queries, exact value matching with configurable tolerance, and unit-only filtering across 13 units in 5 SI domains (distance, mass, time, pressure, velocity).

2. **Unified formula normalization.** We normalize mathematical expressions across whitespace, superscript notation variants, algebraic side-swap, and multiplicative factor reordering. Formula matching is integrated with dimensional operators as parallel branches in a single hybrid retrieval pipeline. No prior system combines formula matching with quantity-aware retrieval.

3. **Federated multi-namespace search with capability negotiation.** A connector architecture supports unified retrieval across filesystem, S3, Qdrant, Neo4j, HDF5, and NetCDF backends without data migration. The retrieval coordinator dynamically negotiates per-backend capabilities, executing only the retrieval branches each namespace supports. Incremental indexing with SHA-256 change detection enables efficient re-indexing of evolving corpora.

4. **HDF5 and NetCDF metadata indexing.** Dedicated connectors extract datasets, attributes, variables, dimensions, and units from HDF5 (via h5py) and NetCDF (via xarray) files. Measurements extracted from scientific file formats pass through the same SI canonicalization pipeline as text-based connectors, enabling cross-format dimensional search.

5. **Multi-hop agentic retrieval.** An LLM-driven query rewriting loop iteratively refines searches over federated scientific data using expand, narrow, and pivot strategies. Each hop executes the full hybrid pipeline with a rewritten query, converging when results stabilize or a maximum hop count is reached.

The remainder of this paper is organized as follows.
